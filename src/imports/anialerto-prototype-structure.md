AniAlerto Prototype Website Structure

The AniAlerto prototype is designed as a web-based platform that includes a public informational website and an administrative management system. The public pages are accessible to all users without logging in, while the administrative module requires authentication by the farm head or administrator.

1. Public Website (No Login Required)
Homepage

The homepage introduces the AniAlerto platform and provides general information about its purpose and benefits for corn farm management.

The homepage contains the following sections:

Header and Navigation Bar

Logo and system name (AniAlerto)

Navigation links such as Home, About AniAlerto, Corn Farming Guide, and Admin Login

Introduction Section

A brief overview explaining that AniAlerto is an SMS-based advisory system designed to assist corn farm workers by sending timely reminders for farm activities such as irrigation, fertilization, pest prevention, and harvesting.

About AniAlerto

Explanation of the system’s goals, including improving communication between farm heads and workers and ensuring that important farming activities are completed on time.

Corn Farming Information

Educational content about corn farming practices, including:

Land preparation

Planting schedule

Fertilization guidelines

Irrigation practices

Pest and disease management

Harvest timing

System Features Overview

Description of key features such as automated SMS reminders, task notifications, and worker response tracking.

Footer Section

Contact information

System credits

Copyright information

2. Administrative Module (Login Required)

The administrative module is accessible only to authorized users such as the farm head or system administrator. This section allows management of farm operations and SMS advisory services.

Admin Login Page

The login page allows the administrator to securely access the system by entering their username and password.

Admin Dashboard

The dashboard provides an overview of system activities and includes quick access to major functions such as batch management, worker registration, advisory scheduling, and message monitoring.

Farm Batch Management

This section allows the administrator to create and manage farm batches. A farm batch represents a group of corn crops with a specific planting date and location. The system uses this information to schedule advisory messages.

Functions include:

Create new farm batch

Edit batch details

View batch schedules

Worker Registration

This module allows the administrator to register and manage farm workers who will receive SMS notifications.

Stored information includes:

Worker name

Mobile phone number

Assigned farm batch

Advisory Message Configuration

This section allows the administrator to configure advisory rules and message templates that will be sent to workers.

Examples include reminders for:

Irrigation

Fertilizer application

Pest monitoring

Harvest preparation

SMS Monitoring

The system records SMS messages sent to workers and tracks their responses.

Workers may reply with simple keywords such as:

DONE – task completed

DELAY – task postponed

HELP – assistance needed

The administrator can view these responses through the monitoring interface.

Reports and Logs

The system generates reports that allow the administrator to review:

Messages sent

Worker responses

Task completion status

These reports help the farm head monitor farm activity and worker compliance.
